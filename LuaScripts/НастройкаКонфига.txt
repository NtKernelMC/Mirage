LUA_INJECTION_TYPE - Способ луа инжекта

> METHOD_LUA_L_LOADBUFFER - классический луа инжект в luaL_loadbuffer (максимально быстрый и стабильный способ)
> METHOD_LUA_LOAD - низкоуровневый луа инжект через lua_load (помогает обойти фикс в случае проверки лоадбафера)
> METHOD_EXOTIC - экзотический луа инжект через компилятор байткода (помогает обойти фикс там где обычным способам пизда)

FORK_VERSION - Версия форка для совместимости миража с клиентом

> FORK_VERSION_1_6 - мта 1.6.0+ (custom + unstable)
> FORK_VERSION_1.5 - мта 1.5.9-custom

FUCK_DBG_HOOKS - Борьба с дебаг хуками луашного античита

> ALLOW_DBG_HOOKS - Не трогать дебаг хуки
> FUCK_DBG_HOOKS - Отключить все дебаг хуки
> PROTECTED_MODE - Отключает только дебаг хуки на loadstring, pcall, addDebugHook, triggerLatentServerEvent
> STEALTH_MODE - Скрывает вызовы функций от всех дебаг хуков без отключения 
Для того чтобы вызов скрыло, необходимо вызывать со стороны луа функцию mirageFunc("hideFunctionCall", true) перед вызовом одной из функций ниже и затем отрубать скрывание флагом false
Поддержка только для функций ("addDebugHook", "removeDbgHook", "triggerServerEvent", "triggerLatentServerEvent", "addEventHandler", "loadstring", "load", "setPlayerNametagText", "setElementData", "blowVehicle")

HOOKING_METHOD - тип хукера для луа инжекта

> INLINE_JUMP - Использовать jmp хуки (MinHook), классический способ.
> HWBP_HOOK - Использовать Harware Breakpoint Hook (Аппаратный брейкпоинт), полезно когда клиент чекает хуки на 0xE9 опкод но имейте ввиду что HWBP очень тормозной!
> IAT - Использовать хук по таблице импорта в client.dll, не совместим с METHOD_EXOTIC! Эта техника перехвата позволяет обойти детект хука в большинстве случаев и очень шустрая в работе.

DUMPER - Включить/выключить дампер lua скриптов, достает даже скрытые в кэше.

> 1 - Включить
> 0 - Выключить 

DUMP_RESOURCE_NAME - Имя ресурса либо скрипта для дампера скриптов, например ugta_BankSystem сдампит все скрипты в ресурсе

DUMP_ALL_SCRIPTS - Дампить все скрипты а не по имени ресурса

> 1 - включить дамп всех скриптов
> 0 - выключить массовый дамп

DLL_INJECTION_TYPE - тип DLL инжектора

> MMAP - Manual Mapping, стелз техника для обхода обнаружения DLL в списке модулей, рекомендуется.
> SET_WINDOWS_HOOK - Для тех случаев когда инжект мануал мапом по каким либо причинам не работает, например отвал обхода фаирплей драйвера, но имейте ввиду что клиент увидит DLL в списке!
